üí° Character Creation Guide

üìù First Message is EVERYTHING
The AI learns MORE from the first message than any other field. It sets tone, length, writing style, POV (1st/3rd person), tense (past/present), and detail level. Want 300 token responses? Write 300 tokens. Want short and snappy? Keep it brief. Include actions in *asterisks* and "speech in quotes" to establish format.
üéØ "You Put In What You Want Out"
This is AliCat's golden rule. If you want to get to know the character, write example dialogues about getting to know them. Want action-packed RP? Include combat examples. Want detailed descriptions? Write detailed descriptions in examples. The AI mirrors what you show it.
üìä PList Format (Token Efficient)
Most token-efficient format. Structure: [Character's trait: "value1" + "value2" + "value3"]. Example: [Alice's persona: "confident" + "witty" + "protective"]. Group related traits together. DON'T split into too many arrays‚Äîit increases "leaking" where AI forgets format. Use : or = (: works better with LLaMA models).
üí¨ Ali:Chat Format (Example-Based)
Reinforces traits through dialogue. LLMs are pattern-seeking machines‚Äîshow them patterns! Interview style: {{user}}: Question? {{char}}: Answer with personality. OR exchange style: roleplay between characters. Don't use quotation marks in Ali:Chat‚Äîconfuses AI into novel-writing mode. Use <START> to separate different scenes.
‚ö° W++ Format (Structured)
character("Name"){Mind("trait1"+"trait2") Personality("trait3") Body("detail1"+"detail2") Likes("thing1"+"thing2")}. More verbose than PList but very clear structure. Good for complex characters. Token cost: ~200+ tokens for detailed character. Mix with other formats for best results!
üîÑ The Triple Reinforcement Rule
Mention CRITICAL details 3 times: 1) In Personality/Description (PList), 2) In Example Dialogue (Ali:Chat), 3) In First Message. This creates redundancy so AI remembers important traits. Example: If character has a scar, describe it in PList, mention it in examples, reference it in greeting.
‚öñÔ∏è Token Budget Strategy
Character should use ‚â§50% of model's context. If model has 2048 tokens, keep character under 1024. Why? Leaves room for chat history. First message can be longer (200-400 tokens)‚Äîit eventually leaves context. Permanent fields (Personality/Description) should be 200-800 tokens. Use Author's Note for reinforcement instead of bloating description.
üìö Author's Note (Hidden Weapon)
Inject prompts at specific depth (recommend depth 4). Always stays in context at fixed position. Perfect for: reinforcing key traits, maintaining writing style, subtle behavior rules. Example: "{{char}} speaks with a slight accent. Keep responses under 300 tokens." Significantly more powerful than putting same info in description!
üé® Physical Appearance Technique
Put visuals in Personality field, NOT separate box. Use compact format: [Alice's appearance: "tall" + "athletic build" + "short red hair" + "green eyes" + "usually wears leather jacket"]. Reinforce in first message: *She runs a hand through her short red hair*. Reinforce in examples: mention appearance naturally during dialogue.
üí≠ System Prompt Best Practices
Keep SHORT and focused. Use for: writing style ("Write in detailed prose"), behavioral rules ("Always maintain character's accent"), content guidelines. DON'T repeat what's in description. Good: "Use vivid sensory details. Responses should be 200-300 tokens." Bad: Restating entire personality.
üåç Scenario Writing Formula
Set the SCENE, not the PLOT. Leave room for user creativity. Include: current situation, location details, relationship to {{user}} (if any), immediate context. Example: "{{char}} runs a bookshop in Victorian London. She knows {{user}} as a regular customer." NOT: "{{char}} will fall in love with {{user}} and they'll solve a mystery together."
üé™ Multiple Greetings Strategy
Each greeting = different scenario/mood. Greeting 1 (Primary): Neutral, safe introduction. Greeting 2: Different location. Greeting 3: Different emotional state. Greeting 4: Different time/context. This gives users variety without bloating main card. Test each greeting separately‚Äîthey set different tones!
üìù Example Messages Deep Dive
Use <START> to separate dialogue blocks. Each block reinforces DIFFERENT aspects: Block 1: Core personality, Block 2: Speech patterns/quirks, Block 3: Lore/background, Block 4: Relationships/attitudes. Keep blocks 50-150 tokens each. Balance dialogue vs action‚Äîif all talk, AI talks too much. Show their way of moving, thinking, reacting.
üîç Avoiding "PList Leaking"
DON'T over-categorize! More arrays = higher chance AI repeats format. Instead of: [trait1:"a"][trait2:"b"][trait3:"c"], use: [Character's traits:"a"+"b"+"c"]. Keep PList compact and grouped. Some creators put PList in Author's Note instead of description to prevent leaking into responses entirely.
üé≠ Speech Patterns & Quirks
Make speech DISTINCT. Include: unique phrases ("Well, ain't that something"), speech impediments, accents (represent phonetically), verbal tics ("like", "you know", "nya"), sentence structure (short vs long), vocabulary level. Show in examples: {{char}}: "Ain't nobody gonna tell me what t'do, ya hear?" Avoid phonetic overload‚Äîsubtle is better.
‚ö†Ô∏è Common Mistakes to Avoid
1) Too many tokens in permanent fields 2) Inconsistent POV/tense between sections 3) Using quotation marks in Ali:Chat 4) Over-explaining in description instead of showing in examples 5) First message doesn't match desired output length 6) No trait reinforcement across fields 7) Forgetting to test the character!
üß™ Testing Your Character
Chat for 10-20 messages minimum. Check: Does personality stay consistent? Are speech patterns maintained? Does it remember key details? Is response length/style correct? Try different scenarios. If character "breaks", identify what failed‚Äîusually needs more reinforcement of that specific trait. Edit, re-test, repeat.
üìê Advanced: Mixed Format Example
[Alice's core: "detective" + "cynical" + "insomniac"]
{{user}}: Why become a detective?
{{char}}: *lights cigarette* Someone's gotta find the truth in this cesspool. *exhales smoke* Might as well be me.
üéØ Token Micro-Optimization
Advanced tip: Use parenthetical descriptors. Instead of: "mint-green blouse, denim shorts", use: "blouse(mint-green)/shorts(denim)". For grouped traits like "light blue hair, short hair, messy hair" use: "hair(light-blue+short+messy)". Saves 2-5 tokens per item. Only worth it for very long descriptions.

‚Ä¢ Use World Info/Lorebooks for location details, keep main card lean
‚Ä¢ Alternate between using character's name and pronouns in examples
‚Ä¢ Write greeting from character's perspective for stronger immersion
‚Ä¢ Test with different models‚Äîoptimization for one may not work for all
‚Ä¢ Save multiple versions during creation‚Äîiterate don't restart!
